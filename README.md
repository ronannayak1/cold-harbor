🏗️ Core Architecture Overview1. The Component BreakdownRather than treating all consumption identically, the pipeline runs three independent forecasting models aligned to distinct consumer behaviors:  📈 Streaming Consumption (streaming): Focuses on passive consumption via the FIRST_WEEK_STREAMING_EQUIVALENT target.  🪙 Digital Track Sales (song): Forecasts transactional downloads via the FIRST_WEEK_SONG_SALE_EQUIVALENT target.  📦 Physical & Direct Product Purchases (product): Captures high-intent monetization via FIRST_WEEK_PRODUCT_SALES.  The final metric is computed via Bottom-Up Reconciliation, ensuring all components sum flawlessly:
  $$\text{Final\_Total\_AE} = \text{Prediction}_{\text{streaming}} + \text{Prediction}_{\text{song}} + \text{Prediction}_{\text{product}}$$ 
  
🔁 Step-by-Step Data & Prediction PipelineStep 1: Temporal Gating & IngestionThe extraction pipeline (singles_archetypes.py) filters raw single-track time-series data to include only historical points occurring strictly before the album drop date ($\text{WEEK\_END\_DATE} < \text{ALBUM\_FIRST\_SALE\_DATE}$). A minimum threshold of 2 observed weeks is strictly enforced.  Step 2: The 5-Step Feature EngineFor valid campaigns, rollout metrics are transformed through five distinct processing blocks:  Curve Stacking: Fits track profiles to a parameterized gamma distribution ($\gamma_{\text{norm}}$) using non-linear least squares optimization. It stacks curves into an absolute trajectory $M_{\text{abs}}(t)$ and a unit-weighted, dimensionless shape trajectory $M_{\text{norm}}(t)$.  Cannibalization Matrix: Evaluates residuals between a track's actual baseline and its expected gamma decay curve at the exact launch week of subsequent singles.  Late-Promo Handling: Bypasses dynamic fitting for late-stage single releases lacking structural history, matching them safely to a rigid template ($a=1.5, t_{\text{peak}}=1.0$).  Carryover Integration: Computes the mathematical integral of $M_{\text{abs}}(t)$ over the release window $[0, 1]$ via trapezoidal aggregation (np.trapezoid) to isolate rollout residual velocity.  Live Catalog Feed: Connects via a live Snowflake link to pull real-time artist catalog data up to the current date. It fits a linear trend slope over the relative time window from week $-8$ to week $-2$.  Step 3: Target Merging & ArchetypingPure shape metrics derived from $M_{\text{norm}}(t)$ pass through a unsupervised MiniBatchKMeans clustering algorithm to assign the campaign to a core rollout archetype. Target-specific historical stats, historical medians, and relative growth interactions are concurrently appended:  $$\text{momentum\_growth\_ratio} = \frac{\text{composite\_peak\_momentum}}{\text{historical\_single\_momentum}}$$Step 4: The Two-Stage Hurdle ArchitectureEach component model executes a dedicated hurdle loop during execution:  Stage 1 (The Gatekeeper): An LGBMClassifier trained via GroupKFold on the 75th percentile ($P_{75}$) of the training target determines if the rollout is "priority". Low-tier failures default to raw carryover + a fallback constant.  Stage 2 (The Magnitude Extrapolator): Priority releases pass to a VotingRegressor ensembling an LGBMRegressor (for non-linear mid-tier patterns) and a regularized linear Ridge pipeline. The linear component permits unconstrained extrapolation on the heavy tail. Targets use a $\log(1+x)$ transform during training and are converted back via np.expm1 during inference.  Step 5: Ensemble ReconciliationComponent forecasts are reconciled through run_bottom_up_ensemble.py. For debut artists, physical product predictions are conditionally capped at 5% of streaming volume to prevent breakout scaling anomalies. All predictions are combined to construct the final synchronized total projection.  

🎯 Production Shortcomings & BlindspotsWhile structurally sound, the pipeline handles several explicit edge cases through post-inference rules:The Sophomore Slump Bias: When an artist's prior era generated record-breaking viral momentum, subsequent rollouts can experience low momentum_growth_ratio figures even if absolute tracking is strong, occasionally requiring a manual safety floor.  Physical Supply Blindness: The models optimize based on historical consumer demand, but remain blind to immediate physical manufacturing execution (e.g., number of vinyl color variants, retail-exclusives, signed SKUs).Debut Artist Product Cold Starts: Breakdown debut artists with no physical sales baseline are highly dependent on shape cluster defaults, which is mitigated via a strict 5% product cap tied directly to streaming velocity.
